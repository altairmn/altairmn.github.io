7.1 DONE Prepare Pascal Parts Ground Truth

Processing Steps:

7.1.1 DONE Reduce Annotations

Done for ALL classes in Pascal PART Stored in data/VOCdevkit/VOC2012/Red_Annotations_Part

7.1.2 DONE Modify Annotations

Here, crop objects from original images. Apply the same procedure to object mask and object part masks. For each file in Red_Annotations_Part, create a file corresponding to each object in that file.

File naming is done by appending the object index in the annotation array to the filename.

This has been done for ALL files in Pascal PART. The data/VOCdevkit/VOC2012/Mod_Annotations_Part contains the annotations for each object in a separate file. All the masks, annotations and images are of size 321x321. The cropped and resized images can be found at data/VOCdevkit/VOC2012/Mod_JPEGImages.

7.1.3 DONE Sketchify Images

Sketchified Image = Object Boundary + Canny Edges + Object Part Boundary

The Canny edge representations of the image are found using the writeCanny.m file. They are stored in data/CannyImages/all. The Canny image representations have been found for ALL objects in the Pascal PART dataset.

10 Day 1

Important Directories :

ROOT : /media/ravikiran/SketchObjPartSegmentation/ switch layer : $ROOT/caffe-switch exper: $ROOT/caffe-switch/exper

Within the exper directory, you'll find all the files in various directories :

  • model contains both pretrained models (for initialization) and training snapshots
  • config contains the solver.prototxt and train.prototxt template files.
  • list contains the list of files required for training
  • data contains training data (HDF5 files)
  • log contains the training log
  • utils contains various files for viewing results, creation of HDF5 data for training, etc.

Within each directory, you'll find folders labeled deeplab_nr, deeplab_ri where nr means no route layer, and ni gives the index of the last subnetwork. Thus n1 means two subnetworks, n2 means three subnetworks, etc.

Wherever you'll see filenames with suffix cxyz that should tell you that it is relevant to the training of classes x, y, z. For example, c23 files are relevant to training for super classes 2, 3. Note that class definitions are given earlier in this document and also in a table later in the document.
